# Ragproject

🚀 Built a Local RAG (Retrieval-Augmented Generation) AI Project using LLMs, Vector Databases, and LangChain!

Excited to share that I successfully developed a fully functional RAG-based AI application as part of my AI Engineering learning journey.

🔹 Tech Stack Used:
• Python
• LangChain
• ChromaDB
• Ollama
• Llama 3
• Recursive Text Splitting
• Vector Embeddings
• RAG Pipeline Architecture

💡 What the project does:
The system loads website/document data, splits it into chunks, converts it into embeddings, stores it inside a vector database, retrieves relevant context, and generates intelligent responses using an LLM.

⚡ Challenges I Faced & Solved:
✅ Faced OpenAI API quota and billing limitations
✅ Encountered multiple LangChain version compatibility issues
✅ Fixed import/module errors in newer LangChain ecosystem
✅ Solved embedding generation problems
✅ Migrated from OpenAI embeddings to local Ollama embeddings
✅ Replaced cloud dependency with Llama 3 for a fully local AI workflow
✅ Configured ChromaDB for efficient vector retrieval

🔧 One of the biggest learnings from this project was understanding how real-world AI systems handle:
• Retrieval pipelines
• Embedding models
• Local vs cloud LLMs
• Vector databases
• Prompt engineering
• Context-aware response generation
